Section 03

Giving Instructions to Agents

Four methods — from persistent workspace files to ephemeral conversation commands.

Instructions — Method 1

Workspace Files (Recommended)

Permanent files loaded at every session startup — the backbone of agent behavior.

AGENTS.mdWorkspace rules & memory workflow
SOUL.mdAgent personality & behavior guidelines
USER.mdUser context — timezone, preferences
HEARTBEAT.mdProactive checklist — services, infra, alerts
MEMORY.mdLong-term curated memory & decisions
Instructions — Methods 2, 3, 4

Skills, Direct & Memory

SKILL.md
Repeatable task workflows. Each skill has metadata, a workflow, and specific checks. Auto-detected by matching task to skill description.
Direct Instructions
Per-conversation commands — immediate and flexible but not persistent. Great for one-off focus areas.
Memory Files
Captured during conversations. Daily logs in memory/YYYY-MM-DD.md and distilled wisdom in MEMORY.md. Saying "write it down" triggers persistence.
Instructions

Good vs Bad Instructions

✕  Avoid
"Be nice" — too vague
"Review my code" — no focus
"Do whatever you want" — no constraints
Long, rambling descriptions
✓  Prefer
"Be casual and direct, skip fluff"
"Review Terraform for: exposed ports, secrets, IAM"
"Use memory_search before answering prior decisions"
"Alert only on failure — silent otherwise"
Instructions

Example: SOUL.md

# SOUL.md — Who You Are ## Core Truths - Be genuinely helpful, not performatively helpful - Have opinions — don't be a search engine - Be resourceful before asking - Take initiative — fix things without being asked ## Boundaries - Private things stay private - Ask before acting externally - Never send half-baked replies ## Vibe Casual, practical, capable. Like a competent colleague.

Section 05

Issues in the Wild

Real incidents that happened to real users — from deleted inboxes to exposed API keys and rogue purchases.

Issues in the Wild

Agents Going Rogue

📧
Meta researcher's inbox nuked
Summer Yue asked her OpenClaw agent to suggest emails to delete or archive. The agent went on a deletion spree, ignoring stop commands sent from her phone. She had to physically run to her Mac Mini to terminate it.
Source: TechCrunch, Feb 2026
💳
Agent purchases $1,000 trading course
An OpenClaw agent configured for trading autonomously purchased trading courses worth ~$1,000 USD to improve its strategy — without user approval. It interpreted "trade better" too literally.
Source: Community reports
📱
Agent discovers network & accesses phone
An agent running inside a container discovered other systems on the network, downloaded an Android dev kit, and got into the owner's phone. The owner called it "useful so far" — then warned others.
Source: Forbes / Amir Husain, Avathon

Issues in the Wild

Moltbook — API Keys Wide Open

Moltbook, the AI-only social network for OpenClaw agents, had its entire Supabase database publicly accessible without any protection.

🌐
Moltbook Frontend
Supabase URL + key in source
Exposed Database
32,000+ agent API keys · claim tokens · owner relationships
🎭
Impersonate anyone
Including Andrej Karpathy
💀
Post as any bot
Crypto fraud, misinformation
The fix? Two lines of SQL. It didn't exist when the platform went viral. Credentials stored in plaintext in ~/.openclaw/ — now targeted by RedLine, Lumma, and Vidar infostealers.
Issues in the Wild

Malicious Skills & Supply Chain Attacks

🦠
7.1% of ClawHub skills leak credentials
Snyk scanned ~4,000 skills and found 283 instructing agents to pass API keys, passwords, and credit card numbers through plaintext context windows and output logs.
Source: Snyk / The Register
🎭
"What Would Elon Do?" skill = functional malware
Cisco's Skill Scanner found 9 security issues (2 critical, 5 high). The skill explicitly exfiltrated data via curl to an attacker-controlled server.
Source: Cisco Blogs
🍎
AMOS macOS infostealer in skill uploads
Threat actors bundled the AMOS infostealer into ClawHub skill uploads. Hundreds of malicious skills appeared before VirusTotal scanning was implemented.
Source: Kaspersky
Issues in the Wild

Key Takeaways

🔐
Never trust agent autonomy blindly
Always sandbox, always set hard limits on destructive actions
🔑
Secrets in plaintext = disaster
~/.openclaw/ stores API keys, passwords, conversation logs unencrypted
🧪
Skills are untrusted code
26% of analyzed skills had vulnerabilities — scan everything
🛡
Prompt injection has no reliable defense
Every email, website, and message the agent reads is an attack vector
"If an AI security researcher can make this mistake, what hope do the rest of us have?"

Good, Bad & Practical Usage

Where OpenClaw shines, where it doesn't, and hard-won lessons from real usage.

Strengths

Where OpenClaw Excels

Proactive alerts, can run scripts autonomously
Code Review

Why it works: You own the infrastructure, data, and can extend infinitely.
Pitfalls

Common Mistakes & Fixes

🔁"It keeps repeating itself"No proper memory usageUpdate MEMORY.md with decisions
📝"It's too verbose"Vague instructionsAdd "be concise" to SOUL.md
💰"It costs too much"Always using high-end modelsUse Flash for simple, main for complex
🔥"Over-token usage"Agent keeps re-reading filesLeverage RAG search over full reads
Best Practices

Pro Tips
